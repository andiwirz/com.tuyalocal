Getting started

You need three things from each device: IP address, Device ID, and Local Key.

The Local Key changes every time you reset or re-pair a device in the Tuya / Smart Life app. If a device stops connecting, get the new key.

Get Device ID & Local Key

Run the tuya-cli wizard on any computer with Node.js ≥ 16:

npx @tuyapi/cli wizard

It logs into the Tuya IoT Platform, links your Smart Life account, and lists every device with its ID and Local Key.

Find the IP address

Use Scan Network in the pairing wizard (UDP broadcast + TCP subnet scan, ~10 s), or check your router's DHCP table. Assign a static IP / DHCP reservation so it never changes.

Smart Plug — DP reference

SettingDefault DPCapabilityTransformR/W
dp_switch 1 On / Off boolean R/W
dp_power 19 Power (W) raw × 0.1 Read
dp_voltage 20 Voltage (V) raw × 0.1 Read
dp_current 18 Current (A) raw × 0.001 Read
dp_energy 0 Energy (kWh) computed from power Read
dp_relay_status 38 Turn On Behavior off / on / memory R/W
dp_fault 0 Fault Alarm raw > 0 = alarm Read
dp_power_factor 0 Power Factor (%) raw as-is Read
dp_countdown 0 Flow action only seconds Write
Energy metering: Most Tuya plugs send DP 17 as a resetting delta that appears frozen locally. Leave dp_energy = 0 (default) — the app computes kWh from power readings instead and persists the total across restarts. Use the Reset energy meter flow action to zero it.

Air Conditioner — DP reference

SettingDefault DPCapabilityNotesR/W
dp_onoff 1 On / Off R/W
dp_target_temp 2 Target Temperature divided by temp_divisor R/W
dp_current_temp 3 Current Temperature divided by temp_divisor Read
dp_mode 4 Mode (picker) Set mode_values to the strings your device sends, e.g. cold,heat,wind,dry,auto R/W
dp_fan_speed 5 Fan Speed (picker) Set fan_speed_values to match your device, e.g. low,middle,high,auto R/W
dp_swing 0 Swing / Louver (picker) Enum type — common: 8 or 15. Set swing_values e.g. off,on R/W
dp_swing_h 0 Horizontal Swing 0 = disabled. Common: 110 R/W
dp_anion 0 Ioniser 0 = disabled. Common: 11 R/W
dp_sleep 0 Sleep Mode 0 = disabled. Common: 9 or 103 R/W
dp_eco 0 ECO Mode 0 = disabled. Common: 6 R/W
dp_child_lock 0 Child Lock 0 = disabled. Common: 7 R/W
dp_countdown_timer 0 Timer 0 = disabled. Common: 10 R/W
dp_countdown_left 0 Timer Remaining 0 = disabled. Common: 11 Read
dp_fault 20 Fault Alarm raw > 0 = alarm active; 0 = disabled Read
Temperature scaling: Some AC units send temperatures multiplied by 10 (e.g. 220 = 22.0 °C). If your displayed temperature is 10× too high, set temp_divisor = 10 in device settings. Auto-detection recognises this during pairing.
Mode, fan & swing values: The allowed strings vary by manufacturer. Check the DP Debug tab while operating the device manually, then update mode_values, fan_speed_values and swing_values in device settings. After saving, restart the app to apply the new picker options.
Auto-detect at pairing: The pairing wizard automatically detects DP 1 (power), DP 2/3 (temperatures), DP 4 (mode), DP 5 (fan speed), DP 11 (ioniser), DP 15 (swing), DP 20 (fault), DP 103 (sleep) and DP 110 (horizontal swing). Only mode_values and fan_speed_values need to be adjusted manually after pairing.

Troubleshooting

SymptomFix
Device stays unavailable Wrong IP, Device ID or Local Key — use Repair to update. Check the Logs tab for the exact error.
Scan finds no devices Tuya devices only accept one TCP connection at a time. If the device is already added, the scan cannot reach it. Enter the IP address manually during pairing instead.
ECONNRESET on every connect Protocol version mismatch — use Auto-detect in the device Repair screen.
Smart Plug values are 10× off Incorrect DP numbers or wrong power scale — adjust in device settings.
Energy (kWh) never updates Set dp_energy = 0 — app computes kWh from power readings.
AC temperature is 10× too high Set temp_divisor = 10 in AC device settings.
AC mode / fan picker shows wrong options Update mode_values / fan_speed_values in device settings to match the strings your device sends, then restart the app.
AC swing picker shows wrong options Update swing_values in device settings (e.g. off,on,updown) to match what the device actually sends. Check the DP Debug tab while toggling swing manually.
Picker still shows old options after edit Restart the Tuya Local app to clear Homey's capability cache.
Spurious water / fault alarm after reconnectBuilt-in debounce suppresses these. If they persist, check the Logs tab for the exact DP value and timing.
