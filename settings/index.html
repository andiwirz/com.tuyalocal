Getting started

You need three things from each device: IP address, Device ID, and Local Key.

The Local Key changes every time you reset or re-pair a device in the Tuya / Smart Life app. If a device stops connecting, get the new key.

Method 1 — Cloud Lookup (recommended)

Use the ☁️ Cloud Lookup tab (above) to fetch Device ID and Local Key directly from the Tuya IoT Platform — no CLI tools needed.

  1. Go to iot.tuya.com → Cloud → Project Management → open your project → Overview
  2. Copy your Access ID / Client ID and Access Secret / Client Secret
  3. Make sure your devices are linked: Devices → Link Tuya App Account → scan QR code with Tuya Smart / Smart Life app
  4. Switch to the ☁️ Cloud Lookup tab, paste your credentials, select your data center, and click Fetch Devices
  5. Use the Copy button to copy Device ID and Local Key

Reporting device issues

If a device does not work as expected or a DP mapping is incorrect, please provide the DP specification from the Cloud Lookup:

  1. Go to the ☁️ Cloud Lookup tab and fetch your devices
  2. Click the device name to open the DP specification
  3. Click Copy DP Table and paste the result into your bug report or community post

This helps the developer identify the correct DP numbers, types, and value ranges for your specific device model.

Find the IP address

Use Scan Network in the pairing wizard (UDP broadcast + TCP subnet scan, ~10 s), or check your router's DHCP table. Assign a static IP / DHCP reservation so it never changes.

Temperature scaling: Some AC units send temperatures multiplied by 10 (e.g. 220 = 22.0 °C). If your displayed temperature is 10× too high, set temp_divisor = 10 in device settings. Auto-detection recognises this during pairing.
Mode, fan & swing values: The allowed strings vary by manufacturer. Check the DP Debug tab while operating the device manually, then update mode_values, fan_speed_values and swing_values in device settings. After saving, restart the app to apply the new picker options.
Auto-detect at pairing: The pairing wizard automatically detects DP 1 (power), DP 2/3 (temperatures), DP 4 (mode), DP 5 (fan speed), DP 11 (ioniser), DP 15 (swing), DP 20 (fault), DP 103 (sleep) and DP 110 (horizontal swing). Only mode_values and fan_speed_values need to be adjusted manually after pairing.

Garage Door — DP reference

Supports four device families. All DPs are auto-detected at pairing time.

SettingDefault DPCapability / UseDevice familyR/W
dp_door_contact 3 garagedoor_closed WOFEA (bool DP 3) · ZC34T (string DP 1: "open"/"closed") · eWeLink (bool DP 2) Read
dp_door_action 0 garagedoor_closed AOSD DP 107 · BoboYun DP 10 — string: opened / closed / opening / closing. 0 = disabled Read
dp_door_control 6 Open/close command WOFEA DP 6 (enum open|close) · AOSD/ZC34T DP 101 (string open|close|stop) Write

door_contact_invert: Enable if the door shows "open" when it is physically closed. This swaps the boolean/string reading from dp_door_contact or dp_door_action.
WOFEA Stop: WOFEA DP 6 only accepts "open" and "close" — sending "stop" is silently ignored. Use the Toggle garage door flow action (dp_switch = 1) to interrupt movement instead.
BoboYun DP 141 alarm: Not auto-detected at pairing (alarm strings don't match WOFEA patterns). Set dp_door_state = 141 manually in device settings after pairing.
Auto-detect summary: WOFEA → contact=3, ctrl=6, switch=1, state=12. ZC34T → contact=1, ctrl=101. AOSD → action=107, ctrl=101, light=105. BoboYun → action=10, open=106, close=107, light=102.

Heat Pump — DP reference

Universal pool / air-water heat pump driver. Covers Phalén Calidi XP, Fairland InverterPlus, Brustec, BWT, CBC, Madimack, Mountfield, Waterco, Aquastrong, Arcelik/Axen, Apricus, ITS 45HD and more. All DPs are auto-detected at pairing time.

SettingTypical DPCapabilityNotesR/W
dp_onoff 1 On / Off DP 101 on Waterco ECO-VS R/W
dp_target_temp 2 Target Temperature Phalén: DP 106 · Waterco: DP 104. Divided by temp_divisor R/W

Temperature divisor: Devices that send temperatures ×10 (e.g. 350 for 35 °C) are auto-detected at pairing. If the displayed temperature is 10× too high, set temp_divisor = 10 in device settings. Mode / preset values: Check the DP Debug tab while switching modes manually to find the exact strings your device sends, then update mode_values and/or preset_values in device settings. Temperature range: Use temp_min, temp_max and temp_step to configure the target temperature slider (defaults: 12–45 °C, step 1).

Curtain Motor — DP reference

Universal curtain / blind motor driver. Supports Zemismart v1 & v2 and any compatible Tuya category-cl curtain motor. All DPs are auto-detected at pairing time.

SettingDefault DPCapability / UseNotesR/W
dp_control 1 windowcoverings_state Enum: open / stop / close R/W
dp_percent_control 2 windowcoverings_set Integer 0–100. 0 = fully closed, 100 = fully open (default convention). Enable invert_position if reversed. R/W

Doorbell — DP reference

Event-driven driver for Tuya video doorbells (Marmitek Buzz LO, Bcom Majic IPBox, Cleverio CD-200 and compatible). The device pushes events over LAN — no polling needed (polling_interval = 0 by default).

SettingDefault DPCapability / UseNotesR/W

dp_motion_event 115 alarm_motion boolean — true starts the motion alarm; auto-clears after motion_reset_seconds Read
dp_alarm_message 0 Ring / motion (base64) DP 185 on some devices. Decodes base64 JSON; fires ring or motion based on cmd field. 0 = disabled Read
dp_motion_switch 134 Motion detection on/off boolean. 0 = disabled. Used by "Enable / Disable motion" flow actions R/W
dp_nightvision 108 Night vision mode number: 0=auto, 1=off, 2=color. 0 = disabled Write
dp_chime_volume 157 Chime volume integer 0–100. 0 = disabled Write
dp_device_volume 160 Speaker volume integer 0–100. 0 = disabled Write
dp_indicator 101 Status LED boolean. 0 = disabled Write
dp_recording 150 Recording on/off boolean. 0 = disabled Write
dp_motion_sensitivity 106 Motion sensitivity number: 0=low, 1=medium, 2=high. 0 = disabled Write
motion_reset_seconds — Motion alarm auto-clear Seconds after which alarm_motion resets to false automatically (default 30) Setting
Seed protection: The ring and motion triggers are suppressed for the very first data packet after each (re)connect. This prevents false triggers when the device sends its initial state on connection.
Base64 alarm message (DP 185): Some cameras encode ring and motion events as a base64 JSON string on DP 185 instead of separate boolean DPs. Set dp_alarm_message = 185 and leave dp_doorbell and dp_motion_event at 0 if your device uses this format. Check the Raw Data tab to identify which format your device uses.
Flow actions: "Enable/Disable motion", "Set night vision", "Set chime volume", and "Set motion sensitivity" all throw an error if the corresponding DP is set to 0. Set the DP number in device settings first.

Troubleshooting

SymptomFix
Device stays unavailable Wrong IP, Device ID or Local Key — use Repair to update. Check the Logs tab for the exact error.
Scan finds no devices Tuya devices only accept one TCP connection at a time. If the device is already added, the scan cannot reach it. Enter the IP address manually during pairing instead.
ECONNRESET on every connect Protocol version mismatch — use Auto-detect in the device Repair screen.
Smart Plug values are 10× off Incorrect DP numbers or wrong power scale — adjust in device settings.
Energy (kWh) never updates Set dp_energy = 0 — app computes kWh from power readings.
AC / Heater temperature is 10× too high Set temp_divisor = 10 in device settings.
AC mode / fan picker shows wrong options Update mode_values / fan_speed_values in device settings to match what your device sends, then restart the app.
AC swing picker shows wrong options Update swing_values (e.g. off,on,updown) to match what the device sends. Check DP Debug while toggling swing manually.
Picker still shows old options after edit Restart the Tuya Local app to clear Homey's capability cache.
Fan direction not detected at pairing Set dp_direction manually in device settings (e.g. DP 63). Values must be forward / reverse.
Light color mode not switching Check the Raw Data tab while switching modes manually to find the exact strings, then update color_mode_white_val / color_mode_color_val in device settings.
Light brightness or color temp range is wrong Adjust brightness_max or color_temp_max in device settings (default 1000; some older devices use 255).
Humidifier water alarm fires on connect Built-in debounce suppresses transient alarms. If they persist, check the Logs tab for the DP value and timing.
Spurious water / fault alarm after reconnect Built-in debounce suppresses these. If they persist, check the Logs tab for the exact DP value and timing.
Garage door shows wrong open/closed state Enable door_contact_invert in device settings to swap the reading. Or set dp_door_contact to the correct DP number using the DP Debug tab.
Garage door Stop action has no effect WOFEA DP 6 ignores "stop" — use Toggle garage door instead (sends a relay pulse on dp_switch). For BoboYun set dp_switch = 103.
Garage door open/close action doesn't work (BoboYun) BoboYun uses separate open/close DPs. Set dp_door_open = 106 and dp_door_close = 107 in device settings.
Garage door alarm never fires (BoboYun) BoboYun alarm DP 141 is not auto-detected. Set dp_door_state = 141 manually in device settings after pairing.
AOSD light not controllable Set dp_light = 105 in device settings (auto-detected at pairing, but verify in DP Debug tab).
Heat pump mode picker does nothing after enabling dp_mode Restart the Tuya Local app — the capability listener is registered at startup. Future changes re-register automatically without a restart.
Heat pump temperature is wrong or 10× off Set temp_divisor = 10 if the device sends temperatures ×10 (e.g. 350 for 35 °C). Auto-detected at pairing for most devices.
Heat pump target temp correct but measured temp is 10× too high Asymmetric DP scaling (e.g. Weau): set current_temp_divisor = 10 in device settings and leave temp_divisor = 1. Set to 0 to fall back to temp_divisor.
Heat pump spurious fault alarm on connect Built-in 30 s grace period suppresses transient faults after reconnect. If alarms persist longer, check the Logs and DP Debug tabs.
Doorbell ring trigger doesn't fire Check that dp_doorbell matches the DP your device uses (common: 136). Use the Raw Data tab while pressing the bell to identify the correct DP. Some devices use base64 on DP 185 — set dp_alarm_message = 185 instead.
Doorbell motion trigger fires on connect Seed protection suppresses the first packet after every (re)connect. If false triggers persist, check the Raw Data tab — the device may be sending motion=true as its idle state.
Doorbell flow action throws "DP is set to 0" error The action requires the corresponding DP to be configured. Set dp_motion_switch, dp_nightvision, dp_chime_volume, or dp_motion_sensitivity to the correct DP number in device settings.
Curtain position slider is inverted (0% = open) Enable invert_position in device settings to swap the direction.
Curtain motor doesn't stop at the right position The motor limit positions are set in the Tuya / Smart Life app (DP 16 border calibration), not in Homey. Run the limit calibration from the original app first.
Curtain tile shows "moving" but motor has stopped The work_state DP may not be resetting to idle on your device. Set dp_work_state = 0 to disable it — the tile state will then follow the control DP echo instead.
Device missing DPs or commands rejected Your device may be using the Standard Instruction Set which hides some DPs. Switch to DP Instruction Set on iot.tuya.com → Devices → your device → Instruction Mode. This exposes all raw DPs with full read/write access.
Cloud Lookup shows fewer DPs than expected Same cause — the Standard Instruction Set limits what the cloud API returns. Switch to DP Instruction mode on iot.tuya.com, then re-fetch in Cloud Lookup.
SET command causes disconnect 1) Refresh Local Key via ☁️ Cloud Lookup. 2) Enable Fire and Forget in device settings. 3) Try Protocol 3.4. 4) Switch to DP Instruction mode on iot.tuya.com.
